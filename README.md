# 🔓 VNC abierto y sin protección: Simulación educativa con Shodan + Remmina

## 🎯 Objetivo del proyecto

Este proyecto muestra de forma educativa cómo un atacante podría detectar e intentar acceder a servicios VNC expuestos en Internet utilizando herramientas como **Shodan** y **Remmina**, todo desde una distribución **Kali Linux**.

> ⚠️ **Este contenido es 100% educativo. No promueve ni apoya actividades ilegales.** El propósito es generar conciencia sobre los riesgos de exponer servicios sin protección.

## 🛠️ Herramientas utilizadas

- 🔍 [Shodan](https://www.shodan.io/)
- 🖥️ Remmina (cliente VNC)
- 🐧 Kali Linux
- 📸 Capturas para cada fase del análisis

## 🧪 Pasos realizados

1. Búsqueda de servicios VNC usando `VNC port:5900` en Shodan
2. Filtrado por país (Argentina)
3. Selección de IPs con puertos abiertos (5900)
4. Conexión con Remmina desde Kali
5. Simulación de fuerza bruta en servicios mal configurados
6. Registro de evidencias visuales

## 🚨 ¿Qué enseñamos con esto?

- Lo fácil que es encontrar equipos vulnerables
- La necesidad de cerrar puertos no usados
- La importancia de cambiar contraseñas por defecto
- El riesgo de no monitorear accesos remotos
